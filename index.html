The Basics of the Web and HTML

How the Web Works

The web is a group of computers that communicate with each other. When a person goes to a web page like www.google.com, their computer sends a HTTP Request to a server.

The server then finds the appropriate HTML document and sends it back to the user's computer where a web browser interprets the page and displays it on the user's screen. This video does a good job of explaining more.

Many types of files can be found on the wed including plain text, HTML, images, videos, music, etc.

HTML

HTML stands for Hypertext Markup Language. This language was invented in the 1990's and is the main type of document on the web.

HTML documents contain text content which describes "what you see" and markup which describes "how it looks". This video gives a good overview.

Tags, Elements, and Attributes

HTML documents and markup are made up of elements. An HTML element refers to everything within a set of opening and closing tags. Tags can bold, italicize, link words or phrases to web pages, images, videos, etc. (with anchor tags), start new paragraphs, and insert images and videos directly to your web page. Some tags known as Void tags do not require closing tags. Types of void tags include image and breakage tags. This video explains the distinction well.

Attributes are properties of HTML elements that provide additional information. These attributes are linked back to CSS and other programming languages to create style to HTML.

Why Computers are Stupid

Computers are stupid because they interpret instructions literally. A small typo or leaving out an end tag can create big problems as computers don't look at the screen and go "oh, that's the problem!". Programmers have to be careful and literal with computers to ensure computers interpret our instructions correctly.

Inline vs. Block Elements

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them. Span is an inline element and div is a block element. This web page explains the difference between span (inline) and div (block) elements well.

Creating a Structured Document with HTML

Developer Tools (in the Browser)

Developer tools in the browser allow you to open up a web page to see how the page was designed and the different code languages used to make the web page work (i.e. HTML, CSS, etc.).

Tree-Like Structure of HTML

HTML elements have elements inside of each other. You can plan these elements out similar to a "family tree". Web pages are set up in boxes and like a family tree, these boxes have boxes inside of them, and boxes inside of those boxes, and so on.

The Relationship between Indented HTML and Boxes

HTML documents are indented to show the box structure of the document. When you read or write HTML code it is indented to illustrate the corresponding box in each section of the page.

Text Editors (for Programming)

Text editors are used to help programmers work faster and with less repetition. Special text editors like Scratchpad and Sublime Text automatically generate closing tags or quotation marks to help the programmer move quicker and avoid costly time mistakes.

Codepen is another editor which allows programmers to use HTML, CSS, and Java languages. It does not auto populate closing tags or quotation marks but Codepen provides a simultaneous mock up on the programmer's work.

Adding CSS Style and HTML Structure

Avoiding Repetition

Avoiding repetition is essential. When programmers don't avoid repetition we leave the door open for errors. Using CSS to avoid repetition allows us to change entire groups of "classes" all at once instead of individually which can open the door for mistakes,

Imagine having 10 or more words bolded in on your web page. If you used the b tag for bold throughout the code and then your client decided they wanted the bold changed to italics, you would have to individually change those b tags to em tags. With CSS, writing code with class elements allows you to make the change all at one time and avoid time consuming mistakes.

CSS

CSS stands for Cascading Style Sheets and it adds style to web pages. Like the tree like structure of HTML, CSS uses a similar type of family tree through inheritance to cascade the properties from a specific element to other elements in the same family tree.

CSS provides all the styles such as font, color, border, padding, positioning, alignment, spacing, box sizing, etc. to a webpage. CSS gives your web page its swagger! Websites like CSS Reference list standard CSS properties, rules, etc. in alphabetical order to assist programmers in locating information quickly and accurately.

Box Sizing and Positioning

HTML elements are boxes. Each box has four components (margins, borders, padding, and content). When creating a webpage it can be difficult to get the size of the box right due to all the box components. There are two techniques to help with sizing. The first is using percentages instead of pixels and the second is setting the box-sizing attribute to border-box for every element. These techniques allow the programmer to get the box sizing right.

Because divs are block elements they take up the entire width of a page. A good way to allow them to sit side-by-side on a web page is to use Flexboxes. Using the rule display: flex; in CSS will put the boxes next to each other to create flexibility in designing your web page.
